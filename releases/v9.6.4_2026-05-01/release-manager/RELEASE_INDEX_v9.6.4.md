## Manifest path policy

Manifest entries are repository-relative paths for the tagged release tree. If assets are uploaded separately as flat GitHub release attachments, use the same byte streams and compare hashes by filename and SHA-256 value.

## Canonical artifacts

The canonical artifacts are the five files in `core/` plus the manifest and checksum files in `manifests/`. Convenience ZIP bundles in `archives/` are not separate canonical authorities.

## Pilot localization artifacts

The pilot localization files are bounded Tier-2 planning materials only. They do not provide legal approval, autonomous deployment authorization, or coercive public-sector authorization. Each starter annex is intentionally non-exhaustive and remains draft status until local legal, policy, technical, stakeholder, and translation review is complete.

## Claim boundary

Claimable:
- Tier 1-3 architectural and run-level conformance where the canon and run artifacts are satisfied.
- Bounded Tier-2 pilot preparation using the pilot localization pack.

Not claimable:
- Tier 4 conformance.
- Full empirical validation.
- Autonomous public-authority deployment readiness.
- Legal approval for Nepal, Vietnam, New Zealand, or any jurisdiction.
- SGP-granted governance authority.
