# Source Hierarchy


## Authority map

| Tier | Artifact family | Controls | Does not control |
|---|---|---|---|
| A | RippleLogic Canon v11.4 | Cascade, equations, gates, PCC, claim boundaries, normative source hierarchy | Empirical validation or legal certification |
| B | SGP v6.4 | Moral-status scalar, protection evidence, protection/authority separation | Cascade order, legal status, or governance authority by itself |
| C | ripple.md Standard v4.4 | Wrapper Decision Notes, assurance records, waiver/triage, scoped conformance claims | Canon cascade replacement |
| D | Agent System v11.4 | Runtime controls, modes, audit, tool-use boundaries | Independent policy authority or legal certification |
| E | Cascade and CSV standards | Compact public summaries subordinate to Canon | Override of Canon text |
| F | Primer, Component Map, guides, validation protocols | Orientation, public explanation, validation planning | Governing Core changes |
| G | Aligners Sheet and examples | Worked-run training and replay examples | Validator, reference calculator, or ProofPack |


For RippleLogic semantics, the Markdown Canon is the governing source. Derived DOCX/PDF renderings are reading formats and do not control if they diverge.

1. RippleLogic Canon v11.4 Markdown source.
2. SGP v6.4 for sentience/moral-status scalar and protection/authority handling.
3. Canonical equations and normative appendices as ranked in Canon Section 0.1.
4. Companion standards and implementation documents, only within their declared scope.
5. Worked examples, templates, release notes, and public explanatory materials.

Package v11.4 is a final core over the v11.4 content line. Exact byte identity is determined by `release/SHA256SUMS.txt`. Component versions remain Canon v11.4 / SGP v6.4 / Agent System v11.4 / ripple.md v4.4 / Primer v3.4 / Aligners Sheet v4.4 unless a future release explicitly changes them.

## v11.4 hardening hierarchy note

The hardening guides and validation scaffolding are companion/support artifacts. They improve implementation discipline, calibration planning, and public usability. They do not override the Canon or SGP. If a hardening guide conflicts with the governing Canon or SGP, the governing source controls until the conflict is resolved in a future versioned release.
