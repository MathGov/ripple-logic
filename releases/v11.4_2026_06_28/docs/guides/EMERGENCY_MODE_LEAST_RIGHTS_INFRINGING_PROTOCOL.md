# Emergency Mode Least-Rights-Infringing Protocol

**Status:** companion hardening guide. Canon controls if conflict occurs.

## Purpose

Emergency Mode is necessary when no option passes RF/NCRC. It must never become a loophole for ordinary welfare compensation. This protocol clarifies how to act under necessity while preserving the Rights Floor as a real floor.

## Core rule

Emergency Mode does not make a rights-failing option aligned. It identifies a time-bounded, least-rights-infringing provisional action when every available option fails normal rights admissibility.

## Selection sequence

1. Declare that no normal option is selectable.
2. Activate Emergency Mode only with independent challenger review unless impossible under time pressure.
3. Construct the rights-violation vector for each option in the Canon's emergency priority order.
4. Choose the option that lexicographically minimizes rights violation depth.
5. Use TRC CVaR only as a secondary tie-break among equal rights vectors.
6. Use RLS only as a final constrained tie-break after rights-vector and TRC comparison.
7. Require sunset, mitigation, monitoring, redress, and post-event review.

## Prohibited uses

Emergency Mode MUST NOT be used to:

- claim ordinary alignment;
- weaken future thresholds;
- avoid redesign;
- replace challenger review;
- rank rights-failing options by ordinary welfare score;
- create precedent for normal-mode selection;
- bypass lawful authority or affected-party protections.

## Emergency Mode record

| Field | Required content |
|---|---|
| emergency_trigger | why no normal option passes |
| options_considered | all known feasible options |
| challenger_status | independent challenger, deferred challenger, or impossible with rationale |
| violation_vectors | per-option rights-violation vector |
| selected_provisional_option | least-rights-infringing option |
| time_limit | fixed sunset or review period |
| harm_cap | maximum allowed harm or escalation threshold |
| mitigation_plan | immediate burden-reduction actions |
| redress_plan | remedy for affected parties |
| review_trigger | condition requiring rerun or termination |

## Plain-language rule

When every path violates the floor, MathGov does not pretend the floor disappeared. It chooses the least-rights-infringing temporary path, records the failure, repairs the harm, and reopens the decision as soon as possible.
