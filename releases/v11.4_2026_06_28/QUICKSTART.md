# Quickstart

Package patch: v11.4 Final Core. Use this as the final documentation/source package before tests, validation packs, and pilots.


## Before you generate or choose options

Do not begin by asking which computed or generated option looks best. Begin by asking whether the option can be responsibly treated as a candidate at all. In this release: computable is not selectable, generated is not grounded, executable is not ethical, and realizable is necessary but not sufficient. The cascade decides whether a candidate has enough reality contact, rights safety, tail safety, structural viability, and residual welfare support to proceed.


## 5-minute path

1. Read `MATHGOV_ESSENTIALS.md`.
2. View `docs/diagrams/cascade_overview.mmd` in a Mermaid viewer.
3. Read `docs/examples/remote_work_policy_example.md`.
4. Read `docs/standards/RippleLogic_Cascade_Standard_v1.0.md`.

## 30-minute path

1. Read `README.md` for release boundaries.
2. Read `REVIEWER_QUICK_START.md` if reviewing correctness, calculability, or GitHub readiness.
2. Read `ARTIFACT_ROLE_MAP.md` and `SOURCE_HIERARCHY.md`.
3. Read `docs/CORE_COMPONENT_MAP.md`.
4. Read `docs/primer/RippleLogic_Foundations_Primer_v3.4.md`.
5. Skim `docs/canon/RippleLogic_v11.4_Canon.md` sections 0-5, 9-11, 17.

## Decision-run path

1. Define options.
2. Fill Reality Grounding.
3. Check RF/NCRC.
4. Check TRC.
5. Check CSV.
6. Score residual welfare with RLS.
7. Produce PCC/Decision Note.
8. Monitor and revise.
