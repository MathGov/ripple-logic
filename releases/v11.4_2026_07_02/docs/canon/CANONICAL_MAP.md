# Canonical Map for RippleLogic Canon v11.4

This file is a reader map for the governing Canon. It does not replace the Canon and does not create new rules.

## Governing sequence

`RG -> RF -> TRC -> CSV -> RLS`

Plain language:

**Ground reality. Protect rights. Bound ruin. Preserve the structure. Score the ripples.**

## Appendix map

The Canon has many appendices because it is a source specification. Public readers should treat them in four groups:

| Appendix group | Function |
|---|---|
| Mathematical and worked-run appendices | equations, replay logic, examples, and calculation discipline |
| Audit and PCC appendices | recordkeeping, flags, proof obligations, and review surfaces |
| Companion-interface appendices | SGP, ripple.md, Agent System, and package alignment boundaries |
| Release-hardening appendices | Appendix AI, Appendix AJ, and Appendix AK, including CSV, public-release hardening, and computability-vs-realizability bridges |

## Review rule

If a short guide, example, workbook, or companion document appears to conflict with the Canon, use the Canon for cascade semantics and use the source hierarchy files to identify which artifact controls the disputed issue.
