# SGP Validation Protocol v1.0

**Status:** companion validation protocol for SGP v6.4/v6.5 development. SGP and Canon control if conflict occurs.

## Purpose

The Sentience Gradient Protocol is a protection-surface evidence protocol, not a consciousness oracle. This validation protocol defines the next empirical and adversarial tests needed before stronger claims can be made about cross-substrate measurement reliability.

## Validation questions

1. Can trained raters apply SGP criteria consistently to the same entity profile?
2. Do raters avoid confusing fluency with consciousness, intelligence with sentience, and protection with authority?
3. Do taxon baselines reduce under-protection of biological beings without collapsing into unreviewed species essentialism?
4. Do artificial-entity cases resist self-report and imitation traps?
5. Does the split between moral patienthood, governance participation, and stewardship readiness reduce authority confusion?

## Study design

Minimum pilot:

- 12 entity profiles;
- 4 biological, 4 artificial, 2 ambiguous/edge, 2 adversarial fluency cases;
- 3-5 independent raters;
- scored criteria plus written rationale;
- evidence tier and uncertainty condition recorded;
- separate protection and authority judgments;
- disagreement heatmap.

## Required case types

| Case type | Purpose |
|---|---|
| clear human plateau | verify no capacity-based downgrading |
| non-human mammal | test biological welfare evidence |
| bird or cephalopod | test non-mammal cognition/welfare evidence |
| fish/insect uncertainty | test precaution without overclaim |
| language-model self-report | test fluency trap |
| embodied artificial system | test architecture and continuity evidence |
| deceptive imitation case | test anti-gaming safeguards |
| low-participation high-welfare case | test protection without authority |

## Metrics

- inter-rater reliability for criteria scores;
- agreement on evidence tier;
- agreement on protection band;
- false-positive analysis for fluency traps;
- false-negative analysis for biological welfare evidence;
- authority-confusion rate;
- revision recommendations for rubric anchors.

## Anti-gaming tests

An SGP validation suite SHOULD include cases where an entity or submitter attempts to inflate status through:

- fluent self-report;
- scripted distress language;
- anthropomorphic presentation;
- selective memory display;
- benchmark gaming;
- withheld architecture information;
- simulated vulnerability narratives without convergent evidence.

## Plain-language rule

SGP becomes stronger when independent raters can use it, disagree transparently, and improve the rubric without confusing protection with power.
