# Claims and Non-Claims

This file is the compact public doorway for the release boundary. It points to `RELEASE_CLAIMS_AND_NON_CLAIMS.md` as the fuller governing statement.

## Claimable now

MathGov Core Release 2026.09 is a Tier 1-3 framework/source release for public review, study, worked-run learning, and pilot design. It specifies the RippleLogic decision architecture inside the MathGov framework using:

`RG -> RF -> TRC -> CSV -> RLS`

Formal shorthand:

`RG/RSG -> RF/NCRC -> TRC -> CSV -> RLS`

Public memory line:

**Ground reality. Protect rights. Bound ruin. Preserve the structure. Score the ripples.**

The package is claimable as a structured, auditable decision-specification and release bundle. The Aligners Sheet is claimable as a worked-run/training companion, not as a validator or reference calculator.

## Not claimable now

This release is not empirical validation, not legal certification, not deployment certification, not ProofPack, not Tier 4, not reference-calculator status, not autonomous governance authority, and not automated moral truth.

## Operational boundary

RLS ranks only options that survive Reality Grounding claim support, the Rights Floor implemented by the Non-Compensatory Rights Constraint, Tail-Risk Constraint, and Containment and Structural Viability. No downstream score may rescue an upstream gate failure.

## Computability and realizability boundary

Claimable: MathGov states that computed, simulated, generated, or model-fluent possibilities do not become decision-authoritative merely because they can be represented. They remain candidates until the cascade establishes grounded claim authority, Rights Floor/NCRC pass, TRC pass, CSV pass or controlled pass, and downstream RLS ranking where applicable.

Not claimable: this release does not implement a realizability-native computer architecture, physics engine, formal-methods kernel, or proof system that prevents unrealizable states from being represented. It is a realizability-aware governance and selectability architecture, not a replacement for future substrate-level work.

## Evidence boundary

RippleLogic improves the structure of ethical decision-making; it does not make poor evidence, missing stakeholders, weak scenario libraries, captured participation, immature measurement, or defective implementation reliable.


## v11.4 Final Core Boundary

v11.4 corrects final documentation-source issues before validation packs and pilots. It does not change the governing cascade and does not convert the framework into a validated decision instrument.


## Complete Ready boundary

Complete Ready means document-package readiness for the declared Tier 1-3 architectural/source release. It does not mean empirical validation, deployment certification, legal certification, ProofPack readiness, Tier 4 readiness, machine-verifiable ecosystem completeness, or automated moral truth.


## Aligners Sheet wording boundary

Do not summarize the worked-run exemplar as “RippleLogic selected Option A.” Accurate wording: both options survive the gates; RLS favors A but non-decisively; the framework refuses deterministic selection; A is separately documented as an authority-selected option within the non-decisive admissible set.
